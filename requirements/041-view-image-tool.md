# Requirement 041: View Image Tool

## Description
The coding agent harness must support a `view_image` tool that allows the agent to examine local image files by sending them to a vision-capable model for analysis. The tool reads an image from disk, encodes it, and automatically sends it to the LLM for visual analysis, returning a textual description of the image contents.

This enables the coding agent to:
- Read text from screenshots
- Analyze diagrams and flowcharts
- Understand UI mockups and design assets
- Inspect visual output for debugging

## Acceptance Criteria
- [ ] `view_image` tool is available in the standard tool set
- [ ] `view_image` tool accepts a `path` parameter (required, string)
- [ ] Tool reads the image file from disk
- [ ] Tool validates the file is a supported image format (PNG, JPEG, WEBP, GIF)
- [ ] Tool enforces a maximum file size (20MB)
- [ ] Tool detects MIME type from file extension, magic bytes, or HTTP content detection
- [ ] Tool returns a base64 data URI in the tool result for vision processing
- [ ] Agent automatically sends the image to a vision-capable model for analysis
- [ ] Agent returns a textual description of the image to the LLM context
- [ ] Clear TUI feedback shows when an image is being analyzed
- [ ] Tool is available in read-only mode (it does not modify any files)
- [ ] Tool call failures are tracked in statistics
- [ ] Error handling for missing files, unsupported formats, and oversized images

## Tool Definition (OpenAI Format)

```json
{
  "type": "function",
  "function": {
    "name": "view_image",
    "description": "View a local image file. Reads the image from disk and sends it to a vision-capable model for analysis. Returns a description of the image contents. Supported formats: PNG, JPEG, WEBP, GIF.",
    "parameters": {
      "type": "object",
      "properties": {
        "path": {
          "type": "string",
          "description": "Path to the image file to view"
        }
      },
      "required": ["path"]
    }
  }
}
```

## Tool Call Format

```json
{
  "id": "call_abc123",
  "type": "function",
  "function": {
    "name": "view_image",
    "arguments": "{\"path\":\"/workspace/screenshot.png\"}"
  }
}
```

### Parameters
- `path`: Path to the image file to view (required, string)
  - Can be an absolute or relative path
  - File must be accessible and readable

## Supported Image Formats

| Format | MIME Type | Extensions |
|--------|-----------|------------|
| PNG | `image/png` | `.png` |
| JPEG | `image/jpeg` | `.jpg`, `.jpeg` |
| WEBP | `image/webp` | `.webp` |
| GIF | `image/gif` | `.gif` (non-animated) |

## MIME Type Detection

The tool detects the image MIME type using multiple strategies:
1. **File extension**: Primary detection method (fast, reliable)
2. **HTTP content type detection**: Fallback using Go's `http.DetectContentType` (examines magic bytes)
3. **Manual magic byte check**: Final fallback for PNG, JPEG, and GIF headers

## File Size Limits

- Maximum file size: **20 MB** (20,971,520 bytes)
- Files exceeding this limit are rejected with a clear error message

## Return Values

### On Success
The tool executor returns the image data internally. The agent then sends the image to a vision model and returns a description:

```
Tool 'view_image' executed successfully.

Image description:
{detailed description generated by the vision model}
```

The internal tool result contains:
- `output`: Brief confirmation with filename, MIME type, and size
- `path`: Path of the image file
- `extra.data_uri`: Base64-encoded data URI for vision processing
- `extra.mime_type`: Detected MIME type
- `extra.size`: File size in bytes

### On Failure

**Missing path parameter:**
```
Tool 'view_image' failed: missing required parameter: path
```

**File not found:**
```
Tool 'view_image' failed: file not found: /path/to/image.png
```

**Unsupported format:**
```
Tool 'view_image' failed: unsupported or unrecognized image format: /path/to/file.txt
```

**File too large:**
```
Tool 'view_image' failed: image file too large: 25165824 bytes (max 20971520 bytes)
```

**Vision analysis failed:**
```
Tool 'view_image' loaded the image but vision analysis failed: {error details}
```

## Processing Flow

1. **Tool Call**: Agent invokes `view_image` with a file path
2. **File Read**: Tool executor reads the image file from disk
3. **Validation**: File size and format are validated
4. **Encoding**: Image is encoded as a base64 data URI
5. **Vision Analysis**: Agent sends the image to a vision-capable LLM with a prompt asking for a detailed description
6. **Result**: Vision model returns a textual description of the image
7. **Context Update**: Description is added to the agent's context for further reasoning

## Vision Model Prompt

When an image is analyzed, the agent sends the following prompt to the vision model:

```
Describe this image in detail. Include any text visible in the image, objects, colors, layout, and any other relevant details.
```

The image is sent using the multi-modal `ContentPart` format with `detail` set to `"auto"`.


### Image Description

```

```

## System Prompt Description

The system prompt includes the following tool description:

```
8. view_image
    Description: View a local image file. Reads the image from disk and sends it to a vision-capable model for analysis. Returns a description of the image contents.
    Parameters:
      - path (string, required): Path to the image file to view
    Supported formats: PNG, JPEG, WEBP, GIF
    How to call: Use view_image when you need to see what's in an image file, read text from screenshots, analyze diagrams, etc.
    Example use case: "What does this screenshot show?", "Read the text in this diagram"
```

## Read-Only Mode

The `view_image` tool is classified as a read-only operation and is available in read-only mode. It only reads files and does not modify the filesystem.

## Use Cases

### Reading Text from Screenshots
```
Agent: "I need to understand what error message is shown in this screenshot"
Agent: Uses view_image with path to error_screenshot.png
Agent: Receives description including the error text
Agent: Uses the error information to debug the issue
```

### Analyzing Diagrams
```
Agent: "I need to understand the architecture from this diagram"
Agent: Uses view_image with path to architecture_diagram.png
Agent: Receives description of the diagram components and relationships
Agent: Uses the information to implement the correct architecture
```

### Inspecting UI Mockups
```
Agent: "I need to verify the UI matches the design mockup"
Agent: Uses view_image with path to design_mockup.png
Agent: Receives detailed description of colors, layout, and elements
Agent: Compares with current implementation
```

### Debugging Visual Output
```
Agent: "The generated chart looks wrong, let me check what it actually shows"
Agent: Uses view_image with path to generated_chart.png
Agent: Receives description revealing the issue (e.g., wrong labels, missing data)
Agent: Fixes the chart generation code
```
